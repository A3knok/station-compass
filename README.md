## サービス概要
駅降車後の「迷子状態」を解消する、改札から出口までに特化したナビゲーションサービスです。
駅構内の構造を基に、最適なルートを案内します
観光客やビジネスマン、上京した方々が大型駅で迷うことなく、
確実に目的の出口に辿り着けるサポートを提供します。

## このサービスへの思い・作りたい理由
大型駅で迷い、目的の出口にたどり着けず冷や汗をかいた経験があります。
特に上京直後や出張中、観光中など「時間の余裕がない」場面では、駅構内での迷子は大きなストレスです。
そこで、旅行者やビジネスマンが迷うことなく目的の出口へ到着できる体験を提供したいと考えました。

## ユーザー層について
### メインターゲット：**観光客、出張で慣れない駅を利用するビジネスパーソン、上京者**
- 観光で東京を訪れる地方在住者
- 出張で慣れない駅を利用するビジネスパーソン  
- 上京したばかりで東京の駅に不慣れな新社会人・学生
### サブターゲット：**通勤・通学で大きな駅を利用する20-30代**
- 転職により新しい駅を利用することになった会社員
- 引越しにより最寄り駅が変わった人
- 大学進学で東京に出てきた学生

ユーザー層は日常的に大型駅を利用するかしないかを大きな軸としました。
効率的な移動にも目を向けて通勤・通学で駅を利用する20-30代もサブターゲットとしています。
20-30代にしている理由は登校時間の幅が広い大学生(=駅利用時間も様々)と転職によって職場が変わる世代(=通勤ルート変更)と考えています。

### 利用シーン：急いでいる時、初めて利用する駅、効率的な移動

## サービスの利用イメージ
### 観光客の場合
東京駅に到着した観光客が、八重洲口に向かいたい時、
改札の位置と目的出口を選ぶだけでルートが表示され、
貴重な観光時間を無駄にせず目的地に到着できます。

### ビジネスマンの場合
出張先の渋谷駅でハチ公出口に向かいたい時、
複雑な駅構内でも確実なルート案内により、
大事な商談に遅刻するリスクを回避できます。

### 上京者の場合
東京生活を始めたばかりの方も、駅構内で迷う不安を感じることなく安心して移動できます。

## ユーザーの獲得について
### MVPリリース
- ターゲット：渋谷駅をよく利用する人
- 目的：情報の蓄積・品質向上
- 課題：初期投稿ユーザー確保、投稿を継続
- 施策(初期投稿ユーザー確保)：自身で渋谷駅の基本ルートを投稿(5件~)、RUNTEQコミュニティで協力依頼
- 施策(投稿継続)：投稿継続促進メッセージ(「渋谷駅で他にも知っているルートはありませんか？」)、投稿数カウント表示、簡易投稿フォームにして投稿ハードルを下げる。

### 本リリース：段階的ユーザー展開

- ターゲット：

フェーズ1：主要駅の日常利用者

  └ 新宿・池袋・東京駅などの情報蓄積

フェーズ2：観光客・出張ビジネスマン

  └ SNS・記事での発信開始

フェーズ3：全国展開

  └ 地方から東京に来る人全般

- 施策：アプリ開発記事の発信、SNS共有機能/OGP設定、さらに「いいね」機能を「役に立った」機能へ変更し社会貢献を感じてもらいます(ボタン押下にて実装)

## サービスの差別化ポイント・推しポイント

### 他サービスとの比較
| **項目** | **他サービス** | **駅からの脱出（当サービス）** |
|----------|----------------|--------------------------------|
| 対象エリア | 屋外、電車間の移動 | **駅構内に特化** |
| ルート基準 |最短ルート、乗り換え効率 | **困りごと解決** |
| 個別対応 | 一律案内 | **状況別ルート提案</br>(雨天屋根付きルートなど)** |

### 当サービスの推しポイント
- 駅構内限定×ユーザー体験型×困りごと別検索」:大分類＋タグ機能で「困りごと別」特化で差別化

- 実際に歩いた人のリアルな体験談をもとに、困りごと（雨天・混雑など）に応じた最適ルートを提供します。
- ランキング形式で利用者の評価を反映しルートの質を向上
